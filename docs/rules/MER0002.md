# MER0002

## Summary

`MER0002` flags broad nested `try/catch` fallback flow inside another `try` block.

## Intent

Broad nested catches hide degraded fallback behavior inside a larger exception flow. That makes the happy path, fallback path, and outer recovery path harder to read together.

## What It Reports

Reports:

```csharp
try {
    if (reader.FieldCount == 1) {
        try {
            data = JsonSerializer.Deserialize<JsonElement>(jsonString);
        }
        catch {
            data = ToJsonElement(rawData);
        }
    }
}
finally {
    _connectionSemaphore.Release();
}
```

```csharp
try {
    foreach (var priceId in priceIds) {
        try {
            await ValidateAsync(priceId);
        }
        catch {
            invalidMappings.Add(priceId);
        }
    }
}
catch (Exception ex) {
    _logger.Error(ex, "Validation failed.");
}
```

## What It Does Not Report

Does not report:

```csharp
try {
}
finally {
    try {
        MetricsRecorder.RecordHttpRequest(method, endpoint, statusCode, elapsed);
    }
    catch (Exception ex) {
        _logger.LogDebug(ex, "Metrics recording failed.");
    }
}
```

```csharp
try {
    try {
        return int.Parse(value);
    }
    catch {
        return 0;
    }
}
catch (Exception ex) {
    return -1;
}
```

```csharp
try {
    try {
        _ = int.Parse(value);
    }
    catch (FormatException) {
        _ = 0;
    }
}
catch (Exception ex) {
}
```

```csharp
try {
    Action parseValue = () => {
        try {
            _ = int.Parse(value);
        }
        catch {
            _ = 0;
        }
    };

    parseValue();
}
catch (Exception ex) {
}
```

## Examples

Incorrect:

```csharp
try {
    try {
        return await primaryClient.ReadAsync(cancellationToken);
    }
    catch (Exception) {
        return await fallbackClient.ReadAsync(cancellationToken);
    }
}
catch (Exception ex) {
    return Failure(ex);
}
```

Correct:

```csharp
try {
    return await ReadWithFallbackAsync(cancellationToken);
}
catch (Exception ex) {
    return Failure(ex);
}
```

### Fallback State Mutation

Incorrect:

```csharp
try {
    foreach (var tenantId in tenantIds) {
        try {
            await SyncTenantAsync(tenantId, cancellationToken);
        }
        catch (Exception ex) {
            failedTenants.Add((tenantId, ex.Message));
        }
    }
}
catch (Exception ex) {
    _logger.Error(ex, "Tenant sync failed.");
}
```

Correct:

```csharp
try {
    foreach (var tenantId in tenantIds) {
        var result = await TrySyncTenantAsync(tenantId, cancellationToken);
        if (!result.Succeeded) {
            failedTenants.Add((tenantId, result.ErrorMessage));
        }
    }
}
catch (Exception ex) {
    _logger.Error(ex, "Tenant sync failed.");
}
```

### Extracted Parser Boundary

Incorrect:

```csharp
try {
    try {
        metadata = JsonSerializer.Deserialize<MetadataDto>(json);
    }
    catch {
        metadata = MetadataDto.Empty;
    }

    await SaveAsync(metadata, cancellationToken);
}
catch (Exception ex) {
    return Failure(ex);
}
```

Correct:

```csharp
try {
    var metadata = ParseMetadataOrDefault(json);
    await SaveAsync(metadata, cancellationToken);
}
catch (Exception ex) {
    return Failure(ex);
}
```

## Preferred Refactor

1. Move the inner risky operation into a helper that handles its own failure path.
2. Return an explicit success/failure result instead of mutating fallback state inline.
3. Keep the outer `try` focused on one failure path.

## Edge Cases

- A nested `try/finally` without an inner catch does not report.
- A nested catch in an outer `finally` block does not report.
- A nested catch that returns or throws from the catch body does not report.
- Specific exception catches such as `catch (FormatException)` do not report.
- A nested catch that only logs does not report.
- A nested catch that logs and also mutates fallback state still reports.
- A nested catch inside a local function or lambda declared in the outer try block does not report.
- Generated code does not report.

## Suppression Guidance

Refactor first. Suppress only when the nested broad catch is genuinely the clearest local shape and extracting it would make the behavior harder to understand.
