# MER0004

## Summary

`MER0004` flags admin and high-risk controller surfaces that rely only on inherited `[Authorize]` instead of declaring an explicit policy.

## Intent

The rule exists because `BaseApiController` proves authentication, not the correct Meridian blast-radius policy. Admin, reports, search, subscriptions, support, tenants, and analytics controllers need explicit policy metadata at the class level or on every HTTP action.

## What It Reports

Reports:

```csharp
[Route("api/admin/users")]
public sealed class AdminUsersController : BaseAdminController {
    [HttpGet]
    public Task<ActionResult<IReadOnlyList<UserDto>>> ListAsync() {
    }
}
```

```csharp
[Route("api/reports")]
public sealed class ReportStatusController : BaseApiController {
    [HttpGet]
    public Task<ActionResult<ReportStatusDto>> GetAsync() {
    }
}
```

Also reports `[AllowAnonymous]` outside approved public, auth, dev, webhook, monitoring, and health surfaces.
The Meridian Analytics event collector is a narrow exception: only the existing `EventsController` ingestion and health actions may declare action-level `[AllowAnonymous]`.

## What It Does Not Report

Does not report:

```csharp
[Route("api/admin/users")]
[Authorize(Policy = AuthorizationPolicies.PlatformAdminOnly)]
public sealed class AdminUsersController : BaseAdminController {
    [HttpGet]
    public Task<ActionResult<IReadOnlyList<UserDto>>> ListAsync() {
    }
}
```

```csharp
[Route("api/admin/platform")]
public sealed class AdminPlatformController : BaseAdminController {
    [HttpGet]
    [Authorize(Policy = AuthorizationPolicies.PlatformAdminOnly)]
    public Task<ActionResult<PlatformHealthDto>> GetHealthAsync() {
    }
}
```

## Examples

Incorrect:

```csharp
[ApiController]
[Route("api/admin/reports")]
public sealed class AdminReportsController : ControllerBase {
    [HttpGet]
    public IActionResult List() => Ok();
}
```

Correct:

```csharp
[ApiController]
[Authorize(Policy = "AdminReader")]
[Route("api/admin/reports")]
public sealed class AdminReportsController : ControllerBase {
    [HttpGet]
    public IActionResult List() => Ok();
}
```

### Action-Level Policy

Incorrect:

```csharp
[Authorize]
[Route("api/subscriptions")]
public sealed class SubscriptionController : ControllerBase {
    [HttpPost("cancel")]
    public Task<IActionResult> CancelAsync() {
        return _workflow.CancelAsync();
    }
}
```

Correct:

```csharp
[Route("api/subscriptions")]
public sealed class SubscriptionController : ControllerBase {
    [Authorize(Policy = "SubscriptionManage")]
    [HttpPost("cancel")]
    public Task<IActionResult> CancelAsync() {
        return _workflow.CancelAsync();
    }
}
```

### Unexpected Anonymous Access

Incorrect:

```csharp
[Route("api/reports")]
public sealed class ReportController : ControllerBase {
    [AllowAnonymous]
    [HttpGet("{id}")]
    public Task<ReportDto> GetAsync(Guid id) {
        return _reader.GetAsync(id);
    }
}
```

Correct:

```csharp
[Route("api/reports")]
public sealed class ReportController : ControllerBase {
    [Authorize(Policy = "ReportRead")]
    [HttpGet("{id}")]
    public Task<ReportDto> GetAsync(Guid id) {
        return _reader.GetAsync(id);
    }
}
```

## Current Rollout

- Live warning scope: `warning` in analyzer-scope backend projects during live analysis and explicit analyzer-enabled lanes
- Inventory command: `rtk pnpm backend:analyzers:inventory -- --diagnostics MER0004 --project apps/backend/Meridian.API/Meridian.API.csproj`
- Fallback loading check: `rtk pnpm backend:analyzers:validate:build -- --diagnostics MER0004`
- Candidate artifact: [Backend custom analyzer staged inventory rollout](docs/analysis/2026-05/2026-05-07-backend-custom-analyzer-staged-inventory.md)

`MER0004` is warning-enabled in committed `.editorconfig` for analyzer-scope live analysis and explicit analyzer-enabled lanes. Normal backend builds still keep analyzers off by default through `apps/backend/Directory.Build.props`. The rule enforces policy presence and obvious anonymous-route drift only; exact policy correctness remains owned by controller policy tests.

## Preferred Refactor

1. Add class-level `[Authorize(Policy = AuthorizationPolicies.X)]` when all actions share the same blast radius.
2. Add action-level policies when a controller mixes read/admin/mutation operations.
3. Keep exact policy-name assertions in `ControllerPolicyRegressionTests`.

## Edge Cases

- Partial controller declarations are evaluated as one controller; a policy declared on one partial satisfies actions declared on another partial.
- Partial controller families with no HTTP actions are ignored.
- A controller with no class-level policy is accepted when every HTTP action has an explicit policy.
- `[AllowAnonymous]` is allowed only in approved public/auth/dev/webhook/monitoring/health locations and the named Analytics event-ingestion actions.
- Class-level `[AllowAnonymous]` on `Meridian.Analytics` `EventsController` still reports so future actions do not inherit public access by accident.

## Suppression Guidance

Default stance: add explicit policy metadata.

A suppression is only justified for a controller whose policy is enforced by a documented custom endpoint filter or middleware boundary that the analyzer cannot see.
