# MER0008

## Summary

`MER0008` flags raw reads of `MERIDIAN_SKIP_*` startup bypass flags outside approved startup guard boundaries.

## Intent

The rule exists because bypass flags can disable validation, background services, Redis, email, Chromium startup, and other production-significant behavior. Raw reads should stay centralized behind `StartupGuards`, tests, or a dedicated typed startup-skip options boundary.

## What It Reports

Reports:

```csharp
var skipEmail = Environment.GetEnvironmentVariable("MERIDIAN_SKIP_EMAIL") == "true";
```

```csharp
var skipBackgroundServices = configuration.GetValue<bool>("MERIDIAN_SKIP_BACKGROUND_SERVICES");
```

```csharp
var skipRedis = configuration["MERIDIAN_SKIP_REDIS"];
```

## What It Does Not Report

Does not report inside `StartupGuards`:

```csharp
if (configuration.GetValue<bool>("MERIDIAN_SKIP_REDIS")) {
    StartupGuards.EnsureTestingOnly(environment);
}
```

Does not report unrelated environment reads:

```csharp
var environment = Environment.GetEnvironmentVariable("ASPNETCORE_ENVIRONMENT");
```

## Examples

Incorrect:

```csharp
var skipGuards = Environment.GetEnvironmentVariable("MERIDIAN_SKIP_STARTUP_GUARDS") == "1";
```

Correct:

```csharp
var skipValidation = StartupGuards.ShouldSkipStartupValidation(environment, configuration);
```

### Raw Bypass Flag in Feature Code

Incorrect:

```csharp
public bool ShouldSkipWarmup() {
    return Environment.GetEnvironmentVariable("MERIDIAN_SKIP_WARMUP") == "1";
}
```

Correct:

```csharp
public bool ShouldSkipWarmup(StartupSkipOptions options) {
    return options.SkipWarmup;
}
```

### Configuration Indexer

Incorrect:

```csharp
public bool SkipAnalytics(IConfiguration configuration) {
    return configuration["MERIDIAN_SKIP_ANALYTICS"] == "true";
}
```

Correct:

```csharp
public bool SkipAnalytics(StartupSkipOptions options) {
    return options.SkipAnalytics;
}
```

`MER0008` launches inventory-only because existing startup/service registration paths need classification before warnings are enabled.

## Preferred Refactor

1. Move raw reads into `StartupGuards` or a typed startup-skip options class.
2. Validate that bypass flags are testing-only or otherwise explicitly constrained.
3. Let feature/service registration code consume the typed/validated boundary instead of reading raw flags.

## Edge Cases

- The rule covers `Environment.GetEnvironmentVariable`, `IConfiguration.GetValue(...)`, and configuration indexer access.
- Literal and compile-time constant string keys beginning with `MERIDIAN_SKIP_` are reported.
- Tests and `StartupGuards` are allowed.

## Suppression Guidance

Default stance: centralize and validate the bypass flag.

A suppression is only justified for a short-lived startup migration where a follow-up removes the raw read.
