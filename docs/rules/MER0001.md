# MER0001

## Summary

`MER0001` flags direct initializer conditionals that branch into object or anonymous-object payload construction.

## Intent

The rule exists because payload construction becomes harder to scan when object-building branches are hidden directly inside initializer members.

## What It Reports

Reports:

```csharp
return new {
    tenant = includeTenant
        ? new {
            slug
        }
        : null
};
```

```csharp
return new SamplePayload {
    Tenant = includeTenant
        ? new TenantPayload {
            Slug = slug
        }
        : null
};
```

## What It Does Not Report

Does not report:

```csharp
var tenant = includeTenant ? slug : null;

return new {
    tenant
};
```

```csharp
return new SamplePayload {
    Tenant = Normalize(includeTenant ? slug : null)
};
```

```csharp
return new SamplePayload {
    OverrideId = isActive ? overrideId : null
};
```

## Examples

Incorrect:

```csharp
var response = new ApiResponse {
    Data = includeDetails ? BuildDetails() : BuildSummary()
};
```

Correct:

```csharp
var payload = includeDetails ? BuildDetails() : BuildSummary();
var response = new ApiResponse {
    Data = payload
};
```

### Anonymous Payload Member

Incorrect:

```csharp
return new {
    plan,
    entitlement = includeEntitlement ? new {
        code,
        remaining
    } : null
};
```

Correct:

```csharp
var entitlement = includeEntitlement
    ? new { code, remaining }
    : null;

return new {
    plan,
    entitlement
};
```

### DTO Payload Member

Incorrect:

```csharp
return new SubscriptionStatusDto {
    Trial = isTrial ? new TrialStatusDto {
        EndsAt = trialEndsAt
    } : null
};
```

Correct:

```csharp
var trial = isTrial
    ? new TrialStatusDto { EndsAt = trialEndsAt }
    : null;

return new SubscriptionStatusDto {
    Trial = trial
};
```

## Preferred Refactor

1. Stage the conditional result in a named local.
2. Build the object or anonymous object from prepared values.
3. Use a helper when multiple members share the same branch decision.

Example:

```csharp
var tenantPayload = includeTenant
    ? new { slug }
    : null;

return new {
    tenant = tenantPayload
};
```

## Edge Cases

- `condition ? null : new { ... }` reports because one branch directly builds payload shape inside the initializer member.
- `condition ? new SomeDto { ... } : null` reports for the same reason.
- Nested anonymous objects inside one branch still report when the outer member itself is the conditional.
- A ternary nested inside an invocation argument does not report.
- Scalar ternaries such as `count == 0 ? 0 : results.Count` do not report.
- Dictionary element initializers are intentionally outside the current contract.

## Suppression Guidance

Default stance: refactor first.

A suppression is only justified when the direct ternary is materially clearer than staging a local and the code is boxed in by an external contract or intentionally temporary.
