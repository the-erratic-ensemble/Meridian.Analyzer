# MER0003

## Summary

`MER0003` flags `[OutputCache]` on endpoints that also carry tenant, entitlement, quota, plan, or explicit policy-sensitive metadata.

## Intent

ASP.NET output-cache hits can skip action execution. That becomes risky when the endpoint also depends on tenant, entitlement, quota, plan, or policy-sensitive request checks.

## What It Reports

Reports:

```csharp
[HttpGet]
[OutputCache(Duration = 120)]
[ConsumeFeature(SubscriptionConstants.FeatureKeys.GeographicSearch)]
public Task<ActionResult<SearchResponse>> SearchAsync() {
}
```

```csharp
[OutputCache(Duration = 120)]
public sealed class ReportController : ControllerBase {
    [HttpGet]
    [TenantScoped]
    public Task<ActionResult<ReportDto>> GetAsync() {
    }
}
```

## What It Does Not Report

Does not report:

```csharp
[HttpGet]
[OutputCache(Duration = 120)]
public Task<ActionResult<ReferenceDataDto>> GetPublicReferenceAsync() {
}
```

## Examples

Incorrect:

```csharp
[Authorize]
[OutputCache]
[HttpGet("api/reports/{tenantId}")]
public Task<ReportDto> GetReportAsync(string tenantId) => _reader.ReadAsync(tenantId);
```

Correct:

```csharp
[Authorize]
[HttpGet("api/reports/{tenantId}")]
public Task<ReportDto> GetReportAsync(string tenantId) => _reader.ReadAsync(tenantId);
```

### Entitlement-Gated Endpoint

Incorrect:

```csharp
[HttpGet("api/search")]
[OutputCache(Duration = 60)]
[ConsumeFeature(SubscriptionConstants.FeatureKeys.GeographicSearch)]
public Task<SearchResponse> SearchAsync(SearchRequest request) {
    return _service.SearchAsync(request);
}
```

Correct:

```csharp
[HttpGet("api/search")]
[ConsumeFeature(SubscriptionConstants.FeatureKeys.GeographicSearch)]
public Task<SearchResponse> SearchAsync(SearchRequest request) {
    return _service.SearchAsync(request);
}
```

### Explicit No-Store

Incorrect:

```csharp
[HttpGet("api/subscriptions/current")]
[OutputCache(Duration = 300)]
[TenantScoped]
public Task<MySubscriptionDto> GetCurrentAsync() {
    return _service.GetCurrentAsync();
}
```

Correct:

```csharp
[HttpGet("api/subscriptions/current")]
[ResponseCache(Location = ResponseCacheLocation.None, NoStore = true)]
[TenantScoped]
public Task<MySubscriptionDto> GetCurrentAsync() {
    return _service.GetCurrentAsync();
}
```

## Preferred Refactor

1. Remove `[OutputCache]` from tenant, entitlement, quota, plan, or policy-sensitive endpoints.
2. Prefer `[ResponseCache(Location = ResponseCacheLocation.None, NoStore = true)]` when the endpoint must explicitly disable shared caching.
3. Introduce `[OutputCache]` only with a cache policy that is clearly safe for every caller who can hit the endpoint.

## Edge Cases

- Class-level `[OutputCache]` reports once when the controller class or any action adds sensitive metadata.
- Method-level `[OutputCache]` reports when the controller class carries sensitive metadata.
- Public/static reference endpoints without sensitive metadata do not report.

## Suppression Guidance

Remove or replace `[OutputCache]` by default. Suppress only when the cached payload is safe across every identity, tenant, entitlement, and plan state that can reach the endpoint.
