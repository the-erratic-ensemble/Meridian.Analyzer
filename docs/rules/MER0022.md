# MER0022

## Summary

`MER0022` flags direct Redis keyspace scans.

## Intent

`IServer.Keys(...)` scans can be expensive and are easy to duplicate. Runtime code should route keyspace scans through a single bounded helper with cancellation, batching, and logging policy.

## What It Reports

Reports direct `StackExchange.Redis.IServer.Keys(...)` invocations outside tests and outside a named `RedisKeyspace*` or `RedisKeyScanner*` helper. Unrelated APIs that happen to expose a `Keys(...)` method are ignored.

## Examples

Incorrect:

```csharp
foreach (var key in server.Keys(pattern: "report:*")) {
    await database.KeyDeleteAsync(key);
}
```

Correct:

```csharp
var scan = _keyspaceScanner.Scan("report:*", pageSize: 250, maxKeys: 5_000, cancellationToken);
foreach (var key in scan.Keys) {
    await database.KeyDeleteAsync(key);
}
```

### Counting Keys

Incorrect:

```csharp
var count = server.Keys(pattern: "tenant:*").Count();
```

Correct:

```csharp
var count = await _keyspaceScanner.CountAsync(
    "tenant:*",
    maxKeys: 10_000,
    cancellationToken);
```

### Maintenance Delete

Incorrect:

```csharp
foreach (var key in server.Keys(pattern: "report-temp:*")) {
    await database.KeyDeleteAsync(key);
}
```

Correct:

```csharp
await _keyspaceMaintenance.DeleteByPatternAsync(
    "report-temp:*",
    pageSize: 250,
    maxKeys: 5_000,
    cancellationToken);
```

## Preferred Refactor

Move scans into one helper that documents caps, batching, cancellation, truncation logging, and operational use cases.
