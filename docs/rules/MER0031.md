# MER0031

## Summary

`MER0031` flags nested `while` loops that only trim a collection back under a count or length threshold.

## Intent

Inline count-trimming loops are housekeeping. When they sit inside another `while` loop, they blur the enclosing loop's main purpose with maintenance logic and make the exit conditions harder to scan.

## What It Reports

Reports:

```csharp
while (await timer.WaitForNextTickAsync(token)) {
    _snapshots.Enqueue(snapshot);

    while (_snapshots.Count > maxSnapshotHistory) {
        _snapshots.TryDequeue(out _);
    }
}
```

```csharp
while (keepRunning) {
    history.Add(entry);

    while (10 < history.Count) {
        history.RemoveAt(0);
    }
}
```

## What It Does Not Report

Does not report:

```csharp
while (history.Count > 10) {
    history.RemoveAt(0);
}
```

```csharp
while (keepRunning) {
    while (queue.Count > 0) {
        var item = queue.Dequeue();
        Process(item);
    }
}
```

```csharp
while (keepRunning) {
    while (history.Count > 10) {
        cursor++;
    }
}
```

## Examples

Incorrect:

```csharp
while (await timer.WaitForNextTickAsync(token)) {
    _recentLatencies.Enqueue(latency);

    while (_recentLatencies.Count > windowSize) {
        _recentLatencies.TryDequeue(out _);
    }
}
```

Correct:

```csharp
while (await timer.WaitForNextTickAsync(token)) {
    _recentLatencies.Enqueue(latency);
    TrimRecentLatencies();
}
```

## Preferred Refactor

1. Move the trimming loop into a named helper such as `TrimRecentLatencies`.
2. Wrap the collection in a bounded queue or ring-buffer abstraction when trimming is core behavior.
3. Keep the enclosing loop focused on its primary polling or processing work.

## Edge Cases

- Generated code does not report.
- The rule only applies to `while` loops nested inside another `while`.
- The condition must compare the same collection's `Count`, `Count()`, or `Length` against a threshold.
- The nested loop body must only perform trim-like mutations on that same collection, such as `TryDequeue` or `RemoveAt`.
