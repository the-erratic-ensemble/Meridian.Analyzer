# MER0005

## Summary

`MER0005` flags admin controller surfaces that do not follow Meridian's structural admin contract.

## Intent

Admin routes are a higher blast-radius surface. Concrete admin controllers should be obvious from name, route, and base type before policy-specific tests run.

## What It Reports

Reports concrete controller classes under admin surfaces, `api/admin` routes, or `Admin*Controller` names when they do not use all three structural signals:

- `Admin*Controller` name
- class-level `api/admin...` route
- `BaseAdminController` inheritance

Partial declarations are evaluated as one controller, so route/base-type metadata does not need to be repeated in every partial file.

## Examples

Incorrect:

```csharp
[Route("api/platform")]
public sealed class PlatformController : ControllerBase {
}
```

Correct:

```csharp
[Route("api/admin/platform")]
public sealed class AdminPlatformController : BaseAdminController {
}
```

### Route Drift

Incorrect:

```csharp
[Route("api/platform/admin")]
public sealed class AdminPlatformController : BaseAdminController {
}
```

Correct:

```csharp
[Route("api/admin/platform")]
public sealed class AdminPlatformController : BaseAdminController {
}
```

### Base Controller Drift

Incorrect:

```csharp
[Route("api/admin/billing")]
public sealed class AdminBillingController : ControllerBase {
}
```

Correct:

```csharp
[Route("api/admin/billing")]
public sealed class AdminBillingController : BaseAdminController {
}
```

## Current Rollout

- Live warning scope: `warning` in analyzer-scope backend projects during live analysis and explicit analyzer-enabled lanes
- Inventory command: `rtk pnpm backend:analyzers:inventory -- --diagnostics MER0005 --project apps/backend/Meridian.API/Meridian.API.csproj`
- Fallback loading check: `rtk pnpm backend:analyzers:validate:build -- --diagnostics MER0005`
- Candidate artifact: [Remaining candidate inventory](docs/analysis/2026-05/2026-05-07-backend-custom-analyzer-remaining-candidates-inventory.md)

`MER0005` is warning-enabled in committed `.editorconfig` for analyzer-scope live analysis and explicit analyzer-enabled lanes. Normal backend builds still keep analyzers off by default through `apps/backend/Directory.Build.props`. Policy-name correctness stays in controller policy tests.

## Preferred Refactor

Rename concrete admin controllers to `Admin*Controller`, inherit `BaseAdminController`, and keep class routes under `api/admin`.
