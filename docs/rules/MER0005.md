# MER0005

## Summary

`MER0005` flags admin controller surfaces that do not follow the package's structural admin contract.

## Intent

Admin routes are a higher blast-radius surface. Concrete admin controllers should be obvious from name, route, and base type before policy-specific tests run.

## What It Reports

Reports concrete controller classes under admin surfaces, `api/admin` routes, or `Admin*Controller` names when they do not use all three structural signals:

- `Admin*Controller` name
- class-level `api/admin...` route
- `AdminControllerBase` inheritance

Partial declarations are evaluated as one controller, so route/base-type metadata does not need to be repeated in every partial file.

## Examples

Incorrect:

```csharp
[Route("api/platform")]
public sealed class PlatformController : ControllerBase {
}
```

Correct:

```csharp
[Route("api/admin/platform")]
public sealed class AdminPlatformController : AdminControllerBase {
}
```

### Route Drift

Incorrect:

```csharp
[Route("api/platform/admin")]
public sealed class AdminPlatformController : AdminControllerBase {
}
```

Correct:

```csharp
[Route("api/admin/platform")]
public sealed class AdminPlatformController : AdminControllerBase {
}
```

### Base Controller Drift

Incorrect:

```csharp
[Route("api/admin/billing")]
public sealed class AdminBillingController : ControllerBase {
}
```

Correct:

```csharp
[Route("api/admin/billing")]
public sealed class AdminBillingController : AdminControllerBase {
}
```

## Preferred Refactor

Rename concrete admin controllers to `Admin*Controller`, inherit `AdminControllerBase`, and keep class routes under `api/admin`.
