# MER0014

## Summary

`MER0014` flags obvious DTO/entity/model ownership drift.

## Intent

Model placement controls coupling. HTTP contracts should stay feature-local, persistence entities should stay in the database entity boundary, and Core models should avoid EF persistence attributes.

## What It Reports

Reports:

- `*Entity` classes outside `Database/Entities` or migrations
- `*Request`, `*Response`, `*Dto`, or `*DTO` classes in `Meridian.Core` or `Meridian.Shared`
- EF persistence attributes on `Meridian.Core` classes

## Examples

Incorrect:

```csharp
namespace Meridian.Shared.Reports;

public sealed class ReportSearchResponseDto {
    public string Title { get; init; } = string.Empty;
}
```

Correct:

```csharp
namespace Meridian.API.Features.Reports.Models;

public sealed class ReportSearchResponseDto {
    public string Title { get; init; } = string.Empty;
}
```

### Entity Placement

Incorrect:

```csharp
namespace Meridian.API.Features.Reports.Models;

public sealed class ReportJobEntity {
    public Guid Id { get; set; }
}
```

Correct:

```csharp
namespace Meridian.Shared.Database.Entities.Reports;

public sealed class ReportJobEntity {
    public Guid Id { get; set; }
}
```

### Persistence Attribute in Core

Incorrect:

```csharp
namespace Meridian.Core.Reports;

[Table("reports")]
public sealed class ReportDefinition {
    public Guid Id { get; init; }
}
```

Correct:

```csharp
namespace Meridian.Core.Reports;

public sealed class ReportDefinition {
    public Guid Id { get; init; }
}
```

## Current Rollout

- Live warning scope: none. The analyzer reports inventory diagnostics at info severity until shared cross-feature contracts are classified.
- Inventory command: `rtk pnpm backend:analyzers:inventory -- --diagnostics MER0014 --project apps/backend/Meridian.Shared/Meridian.Shared.csproj`
- Fallback loading check: `rtk pnpm backend:analyzers:validate:build -- --diagnostics MER0014`
- Candidate artifact: [Remaining candidate inventory](docs/analysis/2026-05/2026-05-07-backend-custom-analyzer-remaining-candidates-inventory.md)

`MER0014` launches inventory-only because shared cross-feature contracts need human classification.

## Preferred Refactor

Move HTTP contracts to feature-local API model folders, keep entities in the database entity boundary, and keep Core persistence-agnostic.
