# MER0016

## Summary

`MER0016` flags ad hoc `System.Text.Json` option construction in runtime code.

## Intent

Shared JSON profiles keep serializer behavior consistent. Per-service option instances create subtle API, persistence, and test drift unless they are named and documented.

## What It Reports

Reports:

- `new JsonSerializerOptions(...)`
- `new JsonDocumentOptions(...)`

The rule allows tests, CLI tools, migrations, `JsonProfiles`, and named `*JsonProfile` or `*JsonOptionsFactory` classes.

## Examples

Incorrect:

```csharp
var options = new JsonSerializerOptions {
    PropertyNamingPolicy = JsonNamingPolicy.CamelCase
};
```

Correct:

```csharp
var options = JsonProfiles.TolerantWeb;
```

### Document Options

Incorrect:

```csharp
using var document = JsonDocument.Parse(payload, new JsonDocumentOptions {
    AllowTrailingCommas = true
});
```

Correct:

```csharp
using var document = JsonDocument.Parse(payload, JsonProfiles.TolerantDocument);
```

### Named Specialized Profile

Incorrect:

```csharp
var payload = JsonSerializer.Serialize(value, new JsonSerializerOptions {
    DefaultIgnoreCondition = JsonIgnoreCondition.WhenWritingNull
});
```

Correct:

```csharp
var payload = JsonSerializer.Serialize(value, ReportPayloadJsonOptions.SerializerOptions);
```

## Preferred Refactor

Move options into `JsonProfiles` or a dedicated factory/profile class and reuse that boundary.
