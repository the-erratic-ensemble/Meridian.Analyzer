# MER0016

## Summary

`MER0016` flags ad hoc `System.Text.Json` option construction in runtime code.

## Intent

Meridian centralizes serializer options through shared JSON profiles. Per-service option instances create subtle API, persistence, and test drift unless they are named and documented.

## What It Reports

Reports:

- `new JsonSerializerOptions(...)`
- `new JsonDocumentOptions(...)`

The rule allows tests, CLI/tools, migrations, `MeridianJsonProfiles`, and named `*JsonProfile` or `*JsonOptionsFactory` classes.

## Examples

Incorrect:

```csharp
var options = new JsonSerializerOptions {
    PropertyNamingPolicy = JsonNamingPolicy.CamelCase
};
```

Correct:

```csharp
var options = MeridianJsonProfiles.TolerantWeb;
```

### Document Options

Incorrect:

```csharp
using var document = JsonDocument.Parse(payload, new JsonDocumentOptions {
    AllowTrailingCommas = true
});
```

Correct:

```csharp
using var document = JsonDocument.Parse(payload, MeridianJsonProfiles.TolerantDocument);
```

### Named Specialized Profile

Incorrect:

```csharp
var payload = JsonSerializer.Serialize(value, new JsonSerializerOptions {
    DefaultIgnoreCondition = JsonIgnoreCondition.WhenWritingNull
});
```

Correct:

```csharp
var payload = JsonSerializer.Serialize(value, ReportPayloadJsonOptions.SerializerOptions);
```
