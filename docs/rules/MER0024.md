# MER0024

## Summary

`MER0024` flags `StringExtensions.IsNullOrEmpty` and `StringExtensions.IsNullOrWhiteSpace` when used inside methods that return queryable/expression predicate surfaces.

## Intent

`StringExtensions` null/whitespace helpers are runtime convenience methods. Using them inside queryable predicate paths risks EF translation failures or provider-specific client-evaluation drift.

## What It Reports

Reports invocation sites (outside tests) where:

- the invocation name is `IsNullOrEmpty` or `IsNullOrWhiteSpace` on a non-`string.` receiver, and
- the containing method returns either `IQueryable...` or `Expression<...>`.

This keeps the rule focused on queryable predicate methods and avoids broad in-memory LINQ noise.

## Examples

Incorrect:

```csharp
public IQueryable<UserEntity> Query(IQueryable<UserEntity> source)
{
    return source.Where(user => !user.Email.IsNullOrWhiteSpace());
}
```

Correct:

```csharp
public IQueryable<UserEntity> Query(IQueryable<UserEntity> source)
{
    return source.Where(user => !string.IsNullOrWhiteSpace(user.Email));
}
```

## Preferred Refactor

Use query-translatable guards (`string.IsNullOrWhiteSpace`, explicit null checks, or shared queryable helper methods) for IQueryable/expression predicates.
