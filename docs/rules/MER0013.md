# MER0013

## Summary

`MER0013` flags source-level layering violations.

## Intent

The package assumes a documented layer dependency flow:

- `Core` must not depend on `Api`, `Infrastructure`, EF Core, or ASP.NET Core.
- `Infrastructure` must not depend on `Api`.
- `Shared` must not depend on host-specific application projects.
- `Analytics` must not use the main application `DbContext` or default repository path.

## What It Reports

Reports:

```csharp
using Infrastructure.Database;
```

```csharp
using Microsoft.EntityFrameworkCore;
```

```csharp
private Infrastructure.Database.AppDbContext? _dbContext;
```

```csharp
private readonly AppDbContext _dbContext;
```

## What It Does Not Report

Does not report:

```csharp
using Shared.Constants;
```

```csharp
using Core.Repositories;
```

## Examples

Incorrect:

```csharp
namespace Core.Reports;

using Infrastructure.Database;

public sealed class ReportPolicy {
    private readonly AppDbContext _dbContext;
}
```

Correct:

```csharp
namespace Core.Reports;

public sealed class ReportPolicy {
    private readonly IReportPolicyReader _reader;
}
```

### Infrastructure Depending on API

Incorrect:

```csharp
namespace Infrastructure.Reports;

using Api.Features.Reports.Models;

public sealed class ReportRepository {
    public Task<ReportDto> ReadAsync(Guid id) {
    }
}
```

Correct:

```csharp
namespace Infrastructure.Reports;

using Core.Reports;

public sealed class ReportRepository {
    public Task<ReportRecord> ReadAsync(Guid id) {
    }
}
```

### Analytics Using API DbContext

Incorrect:

```csharp
namespace Analytics.Services;

using Infrastructure.Database;

public sealed class AnalyticsQueryService {
    private readonly AppDbContext _dbContext;
}
```

Correct:

```csharp
namespace Analytics.Services;

public sealed class AnalyticsQueryService {
    private readonly IClickHouseAnalyticsStore _store;
}
```

## Preferred Refactor

1. Move abstractions inward and implementations outward.
2. Replace forbidden references with Core/Shared abstractions where the dependency is real.
3. Keep analytics on its dedicated storage path instead of adding application `DbContext` usage.

## Edge Cases

- The rule currently analyzes file paths to identify application projects.
- Fully qualified forbidden namespace references are reported even when no `using` directive exists.
- `Analytics` reports `using Infrastructure.Database` and direct `AppDbContext` identifiers.
- Fully dynamic project/package references are outside the Roslyn source rule.

## Suppression Guidance

Move the dependency to the correct layer by default. Suppress only when an explicit architectural decision permits the dependency and the code links to that decision.
