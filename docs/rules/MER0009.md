# MER0009

## Summary

`MER0009` flags async controller actions that do not expose request cancellation and flags `CancellationToken.None` inside controller actions.

## Intent

Controller actions are request boundaries. Async actions should accept a `CancellationToken` so downstream services, repositories, storage calls, and HTTP calls can observe request aborts.

## What It Reports

Reports:

- `[Http*]` controller action methods returning `Task`/`ValueTask` without a `CancellationToken` parameter
- `CancellationToken.None` inside controller action bodies

## Examples

Incorrect:

```csharp
[HttpGet]
public Task<ReportDto> GetAsync() {
    return _reports.ReadAsync(CancellationToken.None);
}
```

Correct:

```csharp
[HttpGet]
public Task<ReportDto> GetAsync(CancellationToken cancellationToken) {
    return _reports.ReadAsync(cancellationToken);
}
```

### Async Action Without Token

Incorrect:

```csharp
[HttpPost]
public async Task<IActionResult> RebuildAsync() {
    await _service.RebuildAsync();
    return Accepted();
}
```

Correct:

```csharp
[HttpPost]
public async Task<IActionResult> RebuildAsync(CancellationToken cancellationToken) {
    await _service.RebuildAsync(cancellationToken);
    return Accepted();
}
```

### Token Lost Mid-Request

Incorrect:

```csharp
[HttpGet]
public Task<ReportDto> GetAsync(CancellationToken cancellationToken) {
    return _reports.ReadAsync(CancellationToken.None);
}
```

Correct:

```csharp
[HttpGet]
public Task<ReportDto> GetAsync(CancellationToken cancellationToken) {
    return _reports.ReadAsync(cancellationToken);
}
```

## Preferred Refactor

Add `CancellationToken cancellationToken` to async actions and forward it through async service/repository calls.
