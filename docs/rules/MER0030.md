# MER0030

## Summary

`MER0030` flags broad per-iteration `try/catch` blocks inside a `while` loop when that loop already sits inside outer `try` flow.

## Intent

Layered exception handling inside long-running loops hides which failures end the loop, which failures are logged and skipped, and which failures escape to the outer control flow. That makes background polling, monitoring, and worker loops harder to review.

## What It Reports

Reports:

```csharp
try {
    while (await timer.WaitForNextTickAsync(token)) {
        try {
            await CollectMetricsAsync(token);
        }
        catch (OperationCanceledException) when (token.IsCancellationRequested) {
        }
        catch (Exception ex) {
            _logger.Error(ex, "Metric collection failed.");
        }
    }
}
catch (OperationCanceledException) when (token.IsCancellationRequested) {
}
```

```csharp
try {
    while (keepRunning) {
        try {
            RefreshLease();
        }
        catch (Exception ex) {
            _logger.Warning(ex, "Lease refresh failed.");
        }
    }
}
catch (Exception ex) {
    _logger.Error(ex, "Worker failed.");
}
```

## What It Does Not Report

Does not report:

```csharp
while (keepRunning) {
    try {
        RefreshLease();
    }
    catch (Exception ex) {
        _logger.Warning(ex, "Lease refresh failed.");
    }
}
```

```csharp
try {
    while (keepRunning) {
        try {
            _ = int.Parse(value);
        }
        catch (FormatException) {
            failures++;
        }
    }
}
catch (Exception ex) {
}
```

```csharp
try {
    while (keepRunning) {
        try {
            return int.Parse(value);
        }
        catch (Exception) {
            return 0;
        }
    }
}
catch (Exception ex) {
    return -1;
}
```

## Examples

Incorrect:

```csharp
try {
    while (await timer.WaitForNextTickAsync(token)) {
        try {
            await PublishSnapshotAsync(token);
        }
        catch (Exception ex) {
            _logger.Error(ex, "Publish failed.");
        }
    }
}
catch (OperationCanceledException) when (token.IsCancellationRequested) {
}
```

Correct:

```csharp
try {
    while (await timer.WaitForNextTickAsync(token)) {
        var result = await TryPublishSnapshotAsync(token);
        if (!result.Succeeded) {
            _logger.Error(result.Exception, "Publish failed.");
        }
    }
}
catch (OperationCanceledException) when (token.IsCancellationRequested) {
}
```

## Preferred Refactor

1. Extract the loop iteration into a helper such as `TryCollectMetricsAsync` or `TryPublishSnapshotAsync`.
2. Return an explicit success or failure result for the iteration instead of mixing loop control and logging in nested catch blocks.
3. Keep the outer `try` focused on loop lifetime and shutdown behavior.

## Edge Cases

- Generated code does not report.
- The rule only applies when the inner `try/catch` is inside a `while` loop and there is an outer `try` outside that `while`.
- Specific exception catches such as `catch (FormatException)` do not report.
- Broad catches that return or throw from the catch body do not report.
- A broad per-iteration catch inside a `while` loop without outer `try` flow does not report.
