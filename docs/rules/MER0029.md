# MER0029

## Summary

`MER0029` flags LINQ or EF fluent chains with eight or more chained calls.

## Intent

Long query chains blur filtering, shaping, ordering, pagination, and materialization into one expression. Splitting them into named intermediate steps makes query intent easier to review and change.

## What It Reports

Reports:

```csharp
return query
    .AsNoTracking()
    .Where(row => row.IsActive)
    .Select(row => row.Id)
    .OrderBy(id => id)
    .ThenBy(id => id.ToString())
    .Skip(10)
    .Take(25)
    .ToListAsync(cancellationToken);
```

```csharp
return values
    .Where(value => value > 0)
    .Select(value => value * 2)
    .OrderBy(value => value)
    .ThenBy(value => value.ToString())
    .Skip(1)
    .Take(10)
    .Distinct()
    .ToArray();
```

## What It Does Not Report

Does not report:

```csharp
return query
    .Where(row => row.IsActive)
    .Select(row => row.Id)
    .OrderBy(id => id)
    .Skip(10)
    .Take(25)
    .ToList();
```

```csharp
return builder
    .SetName(name)
    .SetSlug(slug)
    .EnableCaching()
    .AllowPreview()
    .Build();
```

## Examples

Incorrect:

```csharp
return query
    .Where(row => row.IsActive)
    .Select(row => row.Profile)
    .Where(profile => profile != null)
    .Select(profile => profile!.Id)
    .OrderBy(id => id)
    .Skip(offset)
    .Take(limit)
    .ToArray();
```

Correct:

```csharp
var activeProfiles = query
    .Where(row => row.IsActive)
    .Select(row => row.Profile);

var profileIds = activeProfiles
    .Where(profile => profile != null)
    .Select(profile => profile!.Id);

return profileIds
    .OrderBy(id => id)
    .Skip(offset)
    .Take(limit)
    .ToArray();
```

## Preferred Refactor

1. Break the chain into named intermediate query steps.
2. Separate filtering, shaping, ordering, and materialization when the chain gets long.
3. Keep the final chain short enough to understand in one pass.

## Edge Cases

- Generated code does not report.
- The rule counts extension-method chains on `IEnumerable<T>`, `IQueryable<T>`, and related query-like interfaces.
- A leading `AsNoTracking()` call does not count toward the chain length.
- Custom extension methods on query-like receivers count toward the chain length.
- Non-query fluent APIs do not report.
- The rule reports only once on the outermost query-chain invocation.

## Suppression Guidance

Refactor first. Suppress only when the long fluent chain is already the clearest stable shape and introducing intermediate names would make the query harder to follow.
