# MER0021

## Summary

`MER0021` flags backend production code that bypasses the Serilog logging contract.

## Intent

Backend runtime code standardizes on Serilog. Microsoft `ILogger<T>` and direct `Console` output should stay in framework adapters, hosting edges, CLI/tools, or documented exceptions.

## What It Reports

Reports outside approved framework-edge paths:

- `ILogger<T>` parameter or field types
- `Console.Write...`, `Console.Error.Write...`, and `Console.Out.Write...`

## Examples

Incorrect:

```csharp
Console.WriteLine($"Report {reportId} failed");
```

Correct:

```csharp
_logger.Warning("Report {ReportId} failed", reportId);
```

### Microsoft Logger Injection

Incorrect:

```csharp
public sealed class ReportService {
    private readonly ILogger<ReportService> _logger;

    public ReportService(ILogger<ReportService> logger) {
        _logger = logger;
    }
}
```

Correct:

```csharp
public sealed class ReportService {
    private readonly Serilog.ILogger _logger;

    public ReportService(Serilog.ILogger logger) {
        _logger = logger;
    }
}
```

### Console Error Logging

Incorrect:

```csharp
catch (Exception ex) {
    Console.Error.WriteLine(ex);
}
```

Correct:

```csharp
catch (Exception ex) {
    _logger.Error(ex, "Report generation failed.");
}
```

## Current Rollout

- Live warning scope: `warning` in analyzer-scope backend projects during live analysis and explicit analyzer-enabled lanes
- Inventory command: `rtk pnpm backend:analyzers:inventory -- --diagnostics MER0021 --project apps/backend/Meridian.API/Meridian.API.csproj`
- Fallback loading check: `rtk pnpm backend:analyzers:validate:build -- --diagnostics MER0021`
- Candidate artifact: [Remaining candidate inventory](docs/analysis/2026-05/2026-05-07-backend-custom-analyzer-remaining-candidates-inventory.md)

`MER0021` is warning-enabled in committed `.editorconfig` for analyzer-scope live analysis and explicit analyzer-enabled lanes. Normal backend builds still keep analyzers off by default through `apps/backend/Directory.Build.props`. Middleware, health, hosting, telemetry, and adapter boundaries still need allowlist review.

## Preferred Refactor

Use Serilog directly in runtime services, or isolate Microsoft logging at a framework adapter edge with a named exception.
