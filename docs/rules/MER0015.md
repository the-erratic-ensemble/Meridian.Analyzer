# MER0015

## Summary

`MER0015` flags raw in-memory string-normalization patterns that should usually use Meridian shared string helpers.

## Intent

`Meridian.Shared.Extensions.StringExtensions` centralizes common string handling. In-memory runtime code should prefer helpers such as `HasText`, `TrimToNull`, `OrEmpty`, `TrimUpper`, `TrimLower`, `NormalizePostcode`, and `RedactPostcode`.

## What It Reports

Reports outside tests and `StringExtensions.cs`:

- `string.IsNullOrWhiteSpace(...)`
- `string.IsNullOrEmpty(...)`
- `value ?? string.Empty`
- `Trim()` compared to an empty string
- `Trim().ToUpperInvariant()` or `Trim().ToLowerInvariant()`

Queryable and expression-tree contexts are excluded so EF translation is not accidentally changed by an automated refactor.

Does not report inside `IQueryable` or `Expression<Func<...>>` predicates:

```csharp
public IQueryable<Report> Query(IQueryable<Report> reports) {
    return reports.Where(report => !string.IsNullOrWhiteSpace(report.Name));
}
```

```csharp
Expression<Func<Report, bool>> predicate =
    report => !string.IsNullOrWhiteSpace(report.Name);
```

Still reports in normal in-memory `IEnumerable` code:

```csharp
public IEnumerable<Report> Filter(IEnumerable<Report> reports) {
    return reports.Where(report => !string.IsNullOrWhiteSpace(report.Name));
}
```

## Examples

Incorrect:

```csharp
if (name == null || name.Trim() == string.Empty) {
    return BadRequest();
}
```

Correct:

```csharp
if (name.IsNullOrWhiteSpace()) {
    return BadRequest();
}
```

### Null-to-Empty

Incorrect:

```csharp
var normalized = value ?? string.Empty;
```

Correct:

```csharp
var normalized = value.OrEmpty();
```

### Trim and Case

Incorrect:

```csharp
var featureKey = key.Trim().ToLowerInvariant();
```

Correct:

```csharp
var featureKey = key.TrimLower();
```

### Trim to Optional Value

Incorrect:

```csharp
var displayName = string.IsNullOrWhiteSpace(name) ? null : name.Trim();
```

Correct:

```csharp
var displayName = name.TrimToNull();
```
