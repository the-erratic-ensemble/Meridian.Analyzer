# MER0010

## Summary

`MER0010` flags direct system time, raw `Task.Delay`, and raw timer construction in production runtime code.

## Intent

Meridian has `IMeridianClock` and `TimeProvider` boundaries for deterministic tests and consistent runtime behavior. Direct time and delay calls make retry, polling, and background behavior harder to exercise safely.

## What It Reports

Reports outside approved time/test paths:

- `DateTime.UtcNow`, `DateTime.Now`, `DateTimeOffset.UtcNow`, `DateTimeOffset.Now`
- `Task.Delay(...)`
- `new Timer(...)` or `new PeriodicTimer(...)`

In model/entity/DTO/request/response/value-object paths, passive property or field default initializers are exempt. Method bodies and other runtime behavior in those folders are still reported.

## Examples

### Raw Delay

Incorrect:

```csharp
public Task PauseAsync() {
    return Task.Delay(TimeSpan.FromSeconds(5));
}
```

Correct:

```csharp
public ValueTask PauseAsync(CancellationToken cancellationToken) {
    return _clock.DelayAsync(TimeSpan.FromSeconds(5), cancellationToken);
}
```

### Retry Backoff

Incorrect:

```csharp
public async Task RetryAsync(CancellationToken cancellationToken) {
    await Task.Delay(TimeSpan.FromSeconds(30), cancellationToken);
    await RetryOperationAsync(cancellationToken);
}
```

Correct:

```csharp
public async Task RetryAsync(CancellationToken cancellationToken) {
    await _clock.DelayAsync(TimeSpan.FromSeconds(30), cancellationToken);
    await RetryOperationAsync(cancellationToken);
}
```

### Expiry Checks

Incorrect:

```csharp
public bool IsExpired(DateTimeOffset expiresAt) {
    return DateTimeOffset.UtcNow >= expiresAt;
}
```

Correct:

```csharp
public bool IsExpired(DateTimeOffset expiresAt) {
    return _clock.GetUtcNow() >= expiresAt;
}
```

### Query Windows

Incorrect:

```csharp
public DateTime GetWindowStartUtc() {
    return DateTime.UtcNow.AddDays(-30);
}
```

Correct:

```csharp
public DateTime GetWindowStartUtc() {
    return _timeProvider.GetUtcNow().UtcDateTime.AddDays(-30);
}
```

### Local Time

Incorrect:

```csharp
public DateOnly GetBusinessDate() {
    return DateOnly.FromDateTime(DateTime.Now);
}
```

Correct:

```csharp
public DateOnly GetBusinessDate(TimeZoneInfo businessTimeZone) {
    return DateOnly.FromDateTime(_clock.GetLocalNow(businessTimeZone).DateTime);
}
```

### Raw Timer

Incorrect:

```csharp
public void Start(TimeSpan interval) {
    _timer = new Timer(_ => Refresh(), null, TimeSpan.Zero, interval);
}
```

Correct:

```csharp
public void Start(TimeSpan interval) {
    _timer = _clock.Provider.CreateTimer(
        static state => ((CacheRefresher)state!).Refresh(),
        this,
        TimeSpan.Zero,
        interval);
}
```

### Passive Defaults

No report when a model, DTO, entity, request, response, or value object only has a passive timestamp default:

```csharp
public sealed class ExportStatusDto {
    public DateTime GeneratedAtUtc { get; init; } = DateTime.UtcNow;
}
```

Report when runtime code stamps the DTO directly:

```csharp
public ExportStatusDto CreateStatus() {
    return new ExportStatusDto {
        GeneratedAtUtc = DateTime.UtcNow
    };
}
```

Correct:

```csharp
public ExportStatusDto CreateStatus() {
    return new ExportStatusDto {
        GeneratedAtUtc = _timeProvider.GetUtcNow().UtcDateTime
    };
}
```

## Preferred Refactor

Inject `IMeridianClock` or `TimeProvider`, and use the shared delay/clock abstraction.
