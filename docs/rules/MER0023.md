# MER0023

## Summary

`MER0023` flags detached runtime work without an obvious lifetime or cancellation path.

## Intent

Detached `Task.Run`, fire-and-forget async calls, and broad `CancellationToken.None` usage hide failures, shutdown behavior, concurrency overlap, and request cancellation semantics.

## What It Reports

Reports outside tests, CLI/tools, migrations, analyzer code, and `BackgroundService.ExecuteAsync`:

- `Task.Run(...)`
- fire-and-forget discard assignment such as `_ = PublishAsync()`
- `CancellationToken.None` outside controller actions

Controller-action `CancellationToken.None` is left to `MER0009` to avoid duplicate ownership.

## Examples

Incorrect:

```csharp
_ = Task.Run(() => _renderer.RenderAsync(reportId, CancellationToken.None));
```

Correct:

```csharp
await _backgroundQueue.EnqueueAsync(
    new RenderReportJob(reportId),
    cancellationToken);
```

### Fire-and-Forget Publisher

Incorrect:

```csharp
_ = _eventPublisher.PublishAsync(new ReportCompleted(reportId));
```

Correct:

```csharp
await _eventPublisher.PublishAsync(
    new ReportCompleted(reportId),
    cancellationToken);
```

### Detached Task in Request Flow

Incorrect:

```csharp
[HttpPost("{id}/render")]
public IActionResult Render(Guid id) {
    _ = Task.Run(() => _renderer.RenderAsync(id, CancellationToken.None));
    return Accepted();
}
```

Correct:

```csharp
[HttpPost("{id}/render")]
public async Task<IActionResult> Render(Guid id, CancellationToken cancellationToken) {
    await _renderQueue.EnqueueAsync(new RenderReportJob(id), cancellationToken);
    return Accepted();
}
```

## Preferred Refactor

Use a hosted service, bounded queue, request token, or explicit cancellation and observability path for background work.
