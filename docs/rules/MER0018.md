# MER0018

## Summary

`MER0018` flags raw SQL APIs and interpolated SQL outside persistence code.

## Intent

SQL should live in repositories, migrations, CLI schema tools, or another named persistence path. Raw SQL string APIs are easy to misuse and should be kept visible.

## What It Reports

Reports:

- `FromSqlRaw(...)`
- `ExecuteSqlRaw(...)`
- `SqlQueryRaw(...)`
- interpolated SQL APIs outside persistence code

## Examples

Incorrect:

```csharp
var rows = await dbContext.Reports
    .FromSqlRaw($"select * from reports where tenant_id = '{tenantId}'")
    .ToListAsync(cancellationToken);
```

Correct:

```csharp
var rows = await dbContext.Reports
    .FromSqlInterpolated($"select * from reports where tenant_id = {tenantId}")
    .ToListAsync(cancellationToken);
```

### Raw Command Parameters

Incorrect:

```csharp
await dbContext.Database.ExecuteSqlRawAsync(
    $"delete from reports where tenant_id = '{tenantId}'",
    cancellationToken);
```

Correct:

```csharp
await dbContext.Database.ExecuteSqlInterpolatedAsync(
    $"delete from reports where tenant_id = {tenantId}",
    cancellationToken);
```

### Dynamic Identifier

Incorrect:

```csharp
var rows = await dbContext.Database
    .SqlQueryRaw<ReportRow>($"select * from {tableName}")
    .ToListAsync(cancellationToken);
```

Correct:

```csharp
var tableName = ReportTableNames.ResolveKnownTable(reportKind);
var rows = await _reportSqlReader.ReadFromKnownTableAsync(tableName, cancellationToken);
```

## Preferred Refactor

Prefer interpolated SQL APIs in repositories, keep SQL out of controllers/services, and protect dynamic identifiers with explicit allowlists and tests.
