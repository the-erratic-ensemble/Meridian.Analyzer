# MER0020

## Summary

`MER0020` flags direct repository, DbContext, and EF query work inside controller actions.

## Intent

Controllers should stay focused on HTTP plumbing. Data access and orchestration belong in feature services, facades, or use cases.

## What It Reports

Reports action-body invocations where the receiver is repository-shaped by symbol/type name, or DbContext-shaped by exact receiver name/type and the call is EF-shaped. Generic receiver names such as `context` are ignored unless the type or field/property name indicates a DbContext.

## Examples

Incorrect:

```csharp
[HttpGet]
public Task<Report?> GetAsync(Guid id) {
    return _reportRepository.FindAsync(id);
}
```

Correct:

```csharp
[HttpGet]
public Task<ReportDto?> GetAsync(Guid id, CancellationToken cancellationToken) {
    return _reportService.GetAsync(id, cancellationToken);
}
```

### DbContext Query in Action

Incorrect:

```csharp
[HttpGet]
public Task<List<Report>> ListAsync(CancellationToken cancellationToken) {
    return _dbContext.Reports
        .Where(report => report.TenantId == TenantId)
        .ToListAsync(cancellationToken);
}
```

Correct:

```csharp
[HttpGet]
public Task<IReadOnlyList<ReportDto>> ListAsync(CancellationToken cancellationToken) {
    return _reportService.ListAsync(TenantId, cancellationToken);
}
```

### Repository Mutation in Action

Incorrect:

```csharp
[HttpPost("{id}/cancel")]
public Task CancelAsync(Guid id, CancellationToken cancellationToken) {
    return _reportRepository.CancelAsync(id, cancellationToken);
}
```

Correct:

```csharp
[HttpPost("{id}/cancel")]
public Task CancelAsync(Guid id, CancellationToken cancellationToken) {
    return _reportWorkflow.CancelAsync(id, cancellationToken);
}
```

## Preferred Refactor

Move repository/DbContext/EF work into a feature service or facade and have the action call that service method.
