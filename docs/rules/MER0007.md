# MER0007

## Summary

`MER0007` flags raw environment and configuration reads outside approved configuration boundaries.

## Intent

Runtime feature code should consume validated typed options, startup guards, or provider adapters. Direct `Environment` and `IConfiguration` reads bypass validation and make configuration ownership hard to audit.

## What It Reports

Reports:

- `Environment.GetEnvironmentVariable(...)`
- `configuration[...]`
- `configuration.GetValue(...)`, `GetSection(...)`, `GetRequiredSection(...)`, or `GetConnectionString(...)`

The rule allows tests, startup/configuration/option/provider paths, CLI, tools, and OpenAPI export boundaries.

## Examples

Incorrect:

```csharp
var apiKey = Environment.GetEnvironmentVariable("MERIDIAN_PROVIDER_API_KEY");
```

Correct:

```csharp
public ProviderClient(IOptions<ProviderOptions> options) {
    _apiKey = options.Value.ApiKey;
}
```

### Inline Configuration Lookup

Incorrect:

```csharp
public Task SendAsync(IConfiguration configuration) {
    var apiKey = configuration["ZeptoMail:ApiKey"];
    return _client.SendAsync(apiKey);
}
```

Correct:

```csharp
public sealed class ZeptoMailOptions {
    public string ApiKey { get; init; } = string.Empty;
}
```

### Connection String Lookup

Incorrect:

```csharp
public DbConnection CreateConnection(IConfiguration configuration) {
    var connectionString = configuration.GetConnectionString("PrimaryDatabase");
    return new NpgsqlConnection(connectionString);
}
```

Correct:

```csharp
public sealed class DatabaseOptions {
    public string ConnectionString { get; init; } = string.Empty;
}
```

## Preferred Refactor

Move reads into typed option binders, startup guards, or a named provider adapter. Inject that typed boundary into runtime services.
