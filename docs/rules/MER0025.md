# MER0025

## Summary

`MER0025` flags only empty C# property-pattern braces in backend production code.

## Intent

Codebases using this package can rely on shared extension helpers and plain null-check patterns for ordinary guard cases. Empty property-pattern braces add style drift without carrying meaningful property constraints.

## What It Reports

Reports `is` pattern expressions (outside tests and analyzer-package internals) when the pattern contains an empty property-pattern brace clause, including:

- `value is { } alias`
- `value is not { }`
- `value is ({ }, { })`
- nested empty property patterns such as `value is { Payload: { } }`

## What It Does Not Report

Populated property patterns remain allowed, including:

- `value is { Count: > 0 }`
- `value is { IsTrial: true, TrialEndsAt: not null }`
- `value is { Value.Data: { Count: > 0 } }`

## Examples

### Flagged: Nullable Extraction

Incorrect:

```csharp
if (population is { } populationValue && areaKm2 is > 0) {
    PopulationDensity = populationValue / areaKm2.Value;
}
```

Correct:

```csharp
if (population.TryGetValue(out var populationValue) &&
    areaKm2.TryGetPositiveValue(out var areaValue)) {
    PopulationDensity = populationValue / areaValue;
}
```

### Flagged: Negated Empty Pattern

Incorrect:

```csharp
if (population is not { }) {
    return false;
}
```

Correct:

```csharp
if (!population.TryGetValue(out _)) {
    return false;
}
```

### Flagged: Tuple Centroid Guard

Incorrect:

```csharp
if (localAuthorityCentroid is ({ }, { })) {
    return localAuthorityCentroid;
}
```

Correct:

```csharp
if (localAuthorityCentroid.latitude.TryGetValue(out _) &&
    localAuthorityCentroid.longitude.TryGetValue(out _)) {
    return localAuthorityCentroid;
}
```

Use explicit null checks when the shape does not map cleanly to an existing shared helper.

### Flagged: Nested Empty Property Pattern

Incorrect:

```csharp
if (envelope is { Payload: { } payload }) {
    return payload;
}
```

Correct:

```csharp
if (envelope?.Payload is not null) {
    return envelope.Payload;
}
```

### Allowed: Count Guard With Meaningful Constraint

Allowed:

```csharp
if (reportEntry is { Value.Data: { Count: > 0 } }) {
    return true;
}
```

That pattern encodes an actual property constraint. `MER0025` should not report it.

### Allowed: Domain Property Shape Guard

Allowed:

```csharp
if (feature is { IsTrial: true, TrialEndsAt: not null }) {
    return true;
}
```

That pattern also carries real semantic constraints and remains allowed.

## Preferred Refactor

Prefer the simplest replacement that matches the empty-brace intent:

- `TryGetValue` or `TryGetPositiveValue` for nullable extraction
- direct `is not null` or `!= null` checks for plain reference presence
- explicit tuple-element null checks or nullable helpers for tuple shapes

Do not rewrite populated property patterns just to satisfy style preference. This rule is only about empty brace clauses.
