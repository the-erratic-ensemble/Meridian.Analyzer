# MER0019

## Summary

`MER0019` flags direct `new ProblemDetails` construction inside controller actions.

## Intent

Controller error payloads should flow through shared RFC 7807 mapping utilities so codes, status, detail shape, and extensions stay consistent.

## What It Reports

Reports `new ProblemDetails(...)` inside methods that are both controller methods and `[Http*]` action methods.

## Examples

Incorrect:

```csharp
[HttpGet]
public ActionResult Read() {
    return BadRequest(new ProblemDetails {
        Title = "Invalid report request"
    });
}
```

Correct:

```csharp
[HttpGet]
public ActionResult Read() {
    return Problem(title: "Invalid report request", statusCode: StatusCodes.Status400BadRequest);
}
```

### Not Found Response

Incorrect:

```csharp
[HttpGet("{id}")]
public ActionResult Read(Guid id) {
    return NotFound(new ProblemDetails {
        Title = "Report not found",
        Detail = $"Report {id} was not found."
    });
}
```

Correct:

```csharp
[HttpGet("{id}")]
public ActionResult Read(Guid id) {
    return NotFound(BuildProblemDetails(
        StatusCodes.Status404NotFound,
        "Report not found",
        $"Report {id} was not found."));
}
```

### Validation Response

Incorrect:

```csharp
[HttpPost]
public ActionResult Create(CreateReportRequest request) {
    return BadRequest(new ProblemDetails {
        Title = "Invalid report request",
        Status = StatusCodes.Status400BadRequest
    });
}
```

Correct:

```csharp
[HttpPost]
public ActionResult Create(CreateReportRequest request) {
    return Problem(
        title: "Invalid report request",
        statusCode: StatusCodes.Status400BadRequest);
}
```

## Preferred Refactor

Use controller helpers or a shared ProblemDetails mapping helper from action methods.
