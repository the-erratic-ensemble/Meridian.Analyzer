# MER0027

## Summary

`MER0027` flags overly long boolean `&&` and `||` chains.

## Intent

Large boolean chains are hard to review inline because the reader has to keep too many clauses in working memory at once. Named predicates make the intent legible.

## What It Reports

Reports:

```csharp
return row.CommonName == "station"
    && row.TransportModeName != "bus"
    && row.StopTypeName != "bus"
    && row.TransportModeName != "tram"
    && row.StopTypeName != "tram"
    && row.TransportModeName != "metro";
```

```csharp
return hasExactMatch
    || hasRailName
    || hasTrainName
    || hasStationType
    || hasEntranceType
    || hasFallbackName;
```

## What It Does Not Report

Does not report:

```csharp
return hasExactMatch || hasRailName || hasTrainName || hasStationType;
```

```csharp
var isRailText = hasRailName || hasTrainName;
var isStationLike = hasStationType && !isBus && !isTram;

return isRailText || isStationLike;
```

## Examples

Incorrect:

```csharp
if (a && b && c && d && e && f) {
    return true;
}
```

Correct:

```csharp
var hasRequiredFlags = a && b && c;
var hasOperationalFlags = d && e && f;

if (hasRequiredFlags && hasOperationalFlags) {
    return true;
}
```

## Preferred Refactor

1. Split the chain into named predicates.
2. Group related clauses before combining the groups.
3. If the condition is part of business classification, move it into a named helper.

## Edge Cases

- Generated code does not report.
- The rule reports only the root logical chain.
- Chains with fewer than six clauses do not report.

## Suppression Guidance

Refactor first. Suppress only when the full chain is already the clearest stable shape and additional names would make the condition harder to follow.
